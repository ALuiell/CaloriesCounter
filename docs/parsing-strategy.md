# Parsing Strategy

## Цель

Сделать простой и надежный парсер, который хорошо понимает базовый формат, а не пытается сразу угадать все варианты питания.

## Базовый формат для MVP

Поддерживаем:

```text
рис 100 грамм
курица жареная 150 г
помидор 78
```

## Алгоритм

1. Разбить сообщение по строкам.
2. Очистить текст от лишних пробелов.
3. Для каждой строки выделить число.
4. Считать число весом в граммах.
5. Остальную часть строки считать названием продукта.
6. Нормализовать название:
   - привести к нижнему регистру;
   - убрать лишние пробелы;
   - заменить частые сокращения.
7. Найти точное совпадение или совпадение по alias.
8. Если продукт найден, посчитать значения.
9. Если продукт не найден, вернуть понятную ошибку по этой строке.

## Что не поддерживать в первой версии

- `2 яйца`
- `пол тарелки супа`
- `кусок пиццы`
- `ложка масла`
- составные рецепты без явных ингредиентов

## Обработка ошибок

Если строка не распознана, не нужно ломать все сообщение. Лучше ответить так:

```text
Посчитано:
- рис 100 г
- помидор 78 г

Не удалось распознать:
- курица домашняя большая порция

Попробуйте формат:
название продукта + вес в граммах
Пример: гречка 120 г
```

## Будущие улучшения

- распознавание штук и порций;
- fuzzy search по продуктам;
- уточняющие вопросы;
- пользовательские продукты;
- рецепты и готовые блюда.
